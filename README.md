# 煮理人 — 家庭厨房智能管家

微信小程序 · 情侣/家庭厨房任务协作与菜谱管理工具。

## 项目简介

"煮理人"帮助同居情侣或家庭成员管理厨房相关的日常事务：分配做饭/打扫等家务任务并积分奖励、管理菜谱与冰箱库存、智能推荐晚饭方案，还提供签到、夸夸、排行榜、积分商城等趣味互动功能。

## 技术栈

| 层 | 技术 |
|---|------|
| 前端 | 微信原生小程序（WXML + WXSS + JS） |
| 后端 | 微信云开发（CloudBase） |
| 数据库 | 云开发 JSON 文档数据库 |
| 云函数 | Node.js + wx-server-sdk ~2.6.3 |

## 功能模块

### 家庭管理
- 创建家庭（生成邀请码）或通过邀请码加入
- 家庭成员列表，记录共同生活起始日

### 任务系统
- 发布任务：标题、描述、积分奖励、截止日期
- 任务状态流转：待领取 → 进行中 → 待确认 → 已完成 / 已取消
- 仅创建人可编辑/确认完成任务，任务完成自动发放积分
- 按状态 Tab 筛选

### 菜谱管理
- 快速录入模式（名称+食材必填，步骤选填）
- 展开详细设置（难度、份量、分类标签、步骤、小贴士）
- 菜谱封面 emoji 选择
- 食材逐行录入（名称+数量+单位）
- 仅创建人可编辑

### 冰箱库存
- 食材入库：名称、数量、单位、分类、保质期
- 快过期预警（3 天内标红）
- 与菜谱联动：烹饪时自动消耗对应食材

### 智能推荐
- 根据冰箱库存匹配菜谱，优先推荐能用快过期食材的菜
- 展示匹配度、缺少食材、可清库存项

### 情侣互动
- 每日签到：随机获取 1~5 爱心值，连续签到天数+里程碑徽章
- 今日夸夸：基于日期 hash 从 30+ 条治愈文案库选取
- 签到历史时间轴
- 共同生活天数展示

### 排行榜
- 按周/月/总榜查看家庭积分排名
- 前三名金银铜牌特殊样式
- 底部固定显示当前用户排名

### 积分商城
- 家庭成员可创建兑换商品
- 积分兑换，实时扣减
- emoji 图标选择
- 兑换记录查询

## 数据库集合

| 集合 | 用途 |
|------|------|
| `users` | 用户信息、积分、等级、关联家庭 |
| `families` | 家庭信息、成员、邀请码、起始日 |
| `tasks` | 任务信息、状态、执行人、积分奖励 |
| `recipes` | 菜谱信息、食材、步骤、标签 |
| `fridge_items` | 冰箱库存、数量、单位、保质期 |
| `score_logs` | 积分流水（任务/签到/兑换） |
| `cooking_logs` | 烹饪记录、消耗食材、预警 |
| `check_ins` | 签到记录、连续天数 |
| `shop_items` | 积分商城商品 |
| `redeem_logs` | 兑换记录 |

## 本地开发

### 环境要求

- [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
- Node.js（仅云函数依赖安装需要）

### 启动

1. 克隆项目，用微信开发者工具打开根目录
2. AppID 配置为 `wxe5004372672f61e1`
3. 在微信开发者工具中开通云开发，创建环境
4. 将 `miniprogram/app.js` 中的 `env` 改为你的云环境 ID
5. 右键各云函数文件夹 → "上传并部署：云端安装依赖"
6. 在云开发控制台创建上述数据库集合

### 设计规范

- 全局 CSS 变量定义在 `app.wxss`（主色 #FF8FAB，奶油风暖色调）
- 通用样式类：`.card`、`.btn-primary`、`.flex-row`、`.empty-state` 等
- 云函数统一返回 `{ success: boolean, data?: any, message?: string }`
